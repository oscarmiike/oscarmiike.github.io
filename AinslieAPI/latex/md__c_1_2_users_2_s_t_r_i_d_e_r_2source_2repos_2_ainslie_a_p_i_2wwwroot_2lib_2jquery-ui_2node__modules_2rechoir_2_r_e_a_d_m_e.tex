Look for a module loader associated with the provided file and attempt require it. If necessary, run any setup required to inject it into \href{http://nodejs.org/api/globals.html\#globals_require_extensions}{\texttt{ require.\+extensions}}.

{\ttfamily config} An \href{http://github.com/gulpjs/interpret}{\texttt{ interpret}}-\/like configuration object.

{\ttfamily filepath} A file whose type you\textquotesingle{}d like to register a module loader for.

{\ttfamily cwd} An optional path to start searching for the module required to load the requested file. Defaults to the directory of {\ttfamily filepath}.

{\ttfamily no\+Throw} An optional boolean indicating if the method should avoid throwing.

If calling this method is successful (e.\+g. it doesn\textquotesingle{}t throw), you can now require files of the type you requested natively.

An error with a {\ttfamily failures} property will be thrown if the module loader(s) configured for a given extension cannot be registered.

If a loader is already registered, this will simply return {\ttfamily true}.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__modules_2rechoir_2_r_e_a_d_m_e_autotoc_md11036}{}\doxysubsection{\texorpdfstring{License}{License}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__modules_2rechoir_2_r_e_a_d_m_e_autotoc_md11036}
