The placeholders in the format string are marked by {\ttfamily \%} and are followed by one or more of these elements, in this order\+:


\begin{DoxyItemize}
\item An optional number followed by a {\ttfamily \$} sign that selects which argument index to use for the value. If not specified, arguments will be placed in the same order as the placeholders in the input string.
\item An optional {\ttfamily +} sign that forces to precede the result with a plus or minus sign on numeric values. By default, only the {\ttfamily -\/} sign is used on negative numbers.
\item An optional padding specifier that says what character to use for padding (if specified). Possible values are {\ttfamily 0} or any other character preceded by a `'{\ttfamily (single quote). The default is to pad with \texorpdfstring{$\ast$}{*}spaces\texorpdfstring{$\ast$}{*}.}
\item {\ttfamily An optional}-\/{\ttfamily sign, that causes}sprintf{\ttfamily to left-\/align the result of this placeholder. The default is to right-\/align the result.}
\item {\ttfamily An optional number, that says how many characters the result should have. If the value to be returned is shorter than this number, the result will be padded. When used with the}j$<$tt$>$(\+JSON) type specifier, the padding length specifies the tab size used for indentation.
\item An optional precision modifier, consisting of a.{\ttfamily (dot) followed by a number, that says how many digits should be displayed for floating point numbers. When used with the}g{\ttfamily type specifier, it specifies the number of significant digits. When used on a string, it causes the result to be truncated.}
\item {\ttfamily A type specifier that can be any of\+: \texorpdfstring{$\ast$}{*}}\%{\ttfamily — yields a literal}\%{\ttfamily character \texorpdfstring{$\ast$}{*}}b{\ttfamily — yields an integer as a binary number \texorpdfstring{$\ast$}{*}}c{\ttfamily — yields an integer as the character with that ASCII value \texorpdfstring{$\ast$}{*}}d{\ttfamily or}i{\ttfamily — yields an integer as a signed decimal number \texorpdfstring{$\ast$}{*}}e{\ttfamily — yields a float using scientific notation \texorpdfstring{$\ast$}{*}}u{\ttfamily — yields an integer as an unsigned decimal number \texorpdfstring{$\ast$}{*}}f{\ttfamily — yields a float as is; see notes on precision above \texorpdfstring{$\ast$}{*}}g{\ttfamily — yields a float as is; see notes on precision above \texorpdfstring{$\ast$}{*}}o{\ttfamily — yields an integer as an octal number \texorpdfstring{$\ast$}{*}}s{\ttfamily — yields a string as is \texorpdfstring{$\ast$}{*}}t{\ttfamily — yields}true{\ttfamily or}false{\ttfamily  \texorpdfstring{$\ast$}{*}}T{\ttfamily — yields the type of the argument\texorpdfstring{$<$}{<}sup\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}a href="{}\#fn-\/1"{} name="{}fn-\/ref-\/1"{}\texorpdfstring{$>$}{>}1\texorpdfstring{$<$}{<}/a\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}/sup\texorpdfstring{$>$}{>} \texorpdfstring{$\ast$}{*}}v{\ttfamily — yields the primitive value of the specified argument \texorpdfstring{$\ast$}{*}}x{\ttfamily — yields an integer as a hexadecimal number (lower-\/case) \texorpdfstring{$\ast$}{*}}X{\ttfamily — yields an integer as a hexadecimal number (upper-\/case) \texorpdfstring{$\ast$}{*}}j\`{} — yields a Java\+Script object or array as a JSON encoded string
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m410d85240f84cde6463a6b40e1798031_autotoc_md11984}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m410d85240f84cde6463a6b40e1798031_autotoc_md11984}

You can also swap the arguments. That is, the order of the placeholders doesn\textquotesingle{}t have to match the order of the arguments. You can do that by simply indicating in the format string which arguments the placeholders refer to\+: \begin{DoxyVerb}sprintf('%2$s %3$s a %1$s', 'cracker', 'Polly', 'wants')
\end{DoxyVerb}
 And, of course, you can repeat the placeholders without having to increase the number of arguments.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m410d85240f84cde6463a6b40e1798031_autotoc_md11986}{}\doxysubsection{\texorpdfstring{Named arguments}{Named arguments}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m410d85240f84cde6463a6b40e1798031_autotoc_md11986}
Format strings may contain replacement fields rather than positional placeholders. Instead of referring to a certain argument, you can now refer to a certain key within an object. Replacement fields are surrounded by rounded parentheses -\/ {\ttfamily (} and {\ttfamily )} -\/ and begin with a keyword that refers to a key\+: \begin{DoxyVerb}var user = {
    name: 'Dolly',
}
sprintf('Hello %(name)s', user) // Hello Dolly
\end{DoxyVerb}
 Keywords in replacement fields can be optionally followed by any number of keywords or indexes\+: \begin{DoxyVerb}var users = [
    {name: 'Dolly'},
    {name: 'Molly'},
    {name: 'Polly'},
]
sprintf('Hello %(users[0].name)s, %(users[1].name)s and %(users[2].name)s', {users: users}) // Hello Dolly, Molly and Polly
\end{DoxyVerb}
 Note\+: mixing positional and named placeholders is not (yet) supported\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m410d85240f84cde6463a6b40e1798031_autotoc_md11987}{}\doxysubsection{\texorpdfstring{Computed values}{Computed values}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m410d85240f84cde6463a6b40e1798031_autotoc_md11987}
You can pass in a function as a dynamic value and it will be invoked (with no arguments) in order to compute the value on the fly. \begin{DoxyVerb}sprintf('Current date and time: %s', function() { return new Date().toString() })
\end{DoxyVerb}
