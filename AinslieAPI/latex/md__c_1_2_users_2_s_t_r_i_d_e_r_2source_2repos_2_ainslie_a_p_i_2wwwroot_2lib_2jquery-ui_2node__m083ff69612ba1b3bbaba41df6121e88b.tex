\begin{DoxyVerb}uglifyjs [input files] [options]
\end{DoxyVerb}
 Uglify\+JS can take multiple input files. It\textquotesingle{}s recommended that you pass the input files first, then pass the options. Uglify\+JS will parse input files in sequence and apply any compression options. The files are parsed in the same global scope, that is, a reference from a file to some variable/function declared in another file will be matched properly.

If no input file is specified, Uglify\+JS will read from STDIN.

If you wish to pass your options before the input files, separate the two with a double dash to prevent input files being used as option arguments\+: \begin{DoxyVerb}uglifyjs --compress --mangle -- input.js
\end{DoxyVerb}

Uglify\+JS can generate a source map file, which is highly useful for debugging your compressed Java\+Script. To get a source map, pass {\ttfamily -\/-\/source-\/map -\/-\/output output.\+js} (source map will be written out to {\ttfamily output.\+js.\+map}).

Additional options\+:


\begin{DoxyItemize}
\item `-\/-\/source-\/map "{}filename='$<$\+NAME$>$\textquotesingle{}"{}{\ttfamily to specify the name of the source map. The value of }filename{\ttfamily is only used to set}file\`{} attribute (see \href{https://docs.google.com/document/d/1U1RGAehQwRypUTovF1KRlpiOFze0b-_2gc6fAH0KY0k}{\texttt{ the spec}}) in source map file.
\item `-\/-\/source-\/map "{}root='$<$\+URL$>$\textquotesingle{}"{}\`{} to pass the URL where the original files can be found.
\item `-\/-\/source-\/map "{}url='$<$\+URL$>$\textquotesingle{}"{}{\ttfamily to specify the URL where the source map can be found. Otherwise Uglify\+JS assumes HTTP}X-\/\+Source\+Map{\ttfamily is being used and will omit the }//\# source\+Mapping\+URL=\`{} directive.
\end{DoxyItemize}

For example\+: \begin{DoxyVerb}uglifyjs js/file1.js js/file2.js \
         -o foo.min.js -c -m \
         --source-map "root='http://foo.com/src',url='foo.min.js.map'"
\end{DoxyVerb}
 The above will compress and mangle {\ttfamily file1.\+js} and {\ttfamily file2.\+js}, will drop the output in {\ttfamily foo.\+min.\+js} and the source map in {\ttfamily foo.\+min.\+js.\+map}. The source mapping will refer to {\ttfamily \href{http://foo.com/src/js/file1.js}{\texttt{ http\+://foo.\+com/src/js/file1.\+js}}} and {\ttfamily \href{http://foo.com/src/js/file2.js}{\texttt{ http\+://foo.\+com/src/js/file2.\+js}}} (in fact it will list {\ttfamily \href{http://foo.com/src}{\texttt{ http\+://foo.\+com/src}}} as the source map root, and the original files as {\ttfamily js/file1.\+js} and {\ttfamily js/file2.\+js}).\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m083ff69612ba1b3bbaba41df6121e88b_autotoc_md5085}{}\doxysubsubsection{\texorpdfstring{Composed source map}{Composed source map}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m083ff69612ba1b3bbaba41df6121e88b_autotoc_md5085}
When you\textquotesingle{}re compressing JS code that was output by a compiler such as Coffee\+Script, mapping to the JS code won\textquotesingle{}t be too helpful. Instead, you\textquotesingle{}d like to map back to the original code (i.\+e. Coffee\+Script). Uglify\+JS has an option to take an input source map. Assuming you have a mapping from Coffee\+Script → compiled JS, Uglify\+JS can generate a map from Coffee\+Script → compressed JS by mapping every token in the compiled JS to its original location.

You can pass {\ttfamily -\/-\/comments} to retain certain comments in the output. By default it will keep JSDoc-\/style comments that contain "{}@preserve"{}, "{}@license"{} or "{}@cc\+\_\+on"{} (conditional compilation for IE). You can pass {\ttfamily -\/-\/comments all} to keep all the comments, or a valid Java\+Script regexp to keep only comments that match this regexp. For example {\ttfamily -\/-\/comments /\texorpdfstring{$^\wedge$}{\string^}!/} will keep comments like {\ttfamily /\texorpdfstring{$\ast$}{*}! Copyright Notice \texorpdfstring{$\ast$}{*}/}.

Uglify\+JS has its own abstract syntax tree format; for \href{http://lisperator.net/blog/uglifyjs-why-not-switching-to-spidermonkey-ast/}{\texttt{ practical reasons}} we can\textquotesingle{}t easily change to using the Spider\+Monkey AST internally. Various {\ttfamily compress} transforms that simplify, rearrange, inline and remove code are known to have an adverse effect on debugging with source maps. This is expected as code is optimized and mappings are often simply not possible as some code no longer exists. For highest fidelity in source map debugging disable the Uglify {\ttfamily compress} option and just use {\ttfamily mangle}.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m083ff69612ba1b3bbaba41df6121e88b_autotoc_md5111}{}\doxysubsubsection{\texorpdfstring{Compiler assumptions}{Compiler assumptions}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m083ff69612ba1b3bbaba41df6121e88b_autotoc_md5111}
To allow for better optimizations, the compiler makes various assumptions\+:


\begin{DoxyItemize}
\item {\ttfamily .to\+String()} and {\ttfamily .value\+Of()} don\textquotesingle{}t have side effects, and for built-\/in objects they have not been overridden.
\item {\ttfamily undefined}, {\ttfamily NaN} and {\ttfamily Infinity} have not been externally redefined.
\item {\ttfamily arguments.\+callee}, {\ttfamily arguments.\+caller} and {\ttfamily Function.\+prototype.\+caller} are not used.
\item The code doesn\textquotesingle{}t expect the contents of {\ttfamily Function.\+prototype.\+to\+String()} or {\ttfamily Error.\+prototype.\+stack} to be anything in particular.
\item Getting and setting properties on a plain object does not cause other side effects (using {\ttfamily .watch()} or {\ttfamily Proxy}).
\item Object properties can be added, removed and modified (not prevented with {\ttfamily Object.\+define\+Property()}, {\ttfamily Object.\+define\+Properties()}, {\ttfamily Object.\+freeze()}, {\ttfamily Object.\+prevent\+Extensions()} or {\ttfamily Object.\+seal()}). 
\end{DoxyItemize}
