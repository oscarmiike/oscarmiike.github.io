 For reading user input from stdin.

Similar to the {\ttfamily readline} builtin\textquotesingle{}s {\ttfamily question()} method, but with a few more features.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4139}{}\doxysection{\texorpdfstring{USAGE}{USAGE}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4139}

The callback gets called with either the user input, or the default specified, or an error, as {\ttfamily callback(error, result, is\+Default)} node style.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4140}{}\doxysection{\texorpdfstring{OPTIONS}{OPTIONS}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4140}
Every option is optional.


\begin{DoxyItemize}
\item {\ttfamily prompt} What to write to stdout before reading input.
\item {\ttfamily silent} Don\textquotesingle{}t echo the output as the user types it.
\item {\ttfamily replace} Replace silenced characters with the supplied character value.
\item {\ttfamily timeout} Number of ms to wait for user input before giving up.
\item {\ttfamily default} The default value if the user enters nothing.
\item {\ttfamily edit} Allow the user to edit the default value.
\item {\ttfamily terminal} Treat the output as a TTY, whether it is or not.
\item {\ttfamily input} Readable stream to get input data from. (default {\ttfamily process.\+stdin})
\item {\ttfamily output} Writeable stream to write prompts to. (default\+: {\ttfamily process.\+stdout})
\end{DoxyItemize}

If silent is true, and the input is a TTY, then read will set raw mode, and read character by character.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4141}{}\doxysection{\texorpdfstring{COMPATIBILITY}{COMPATIBILITY}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4141}
This module works sort of with node 0.\+6. It does not work with node versions less than 0.\+6. It is best on node 0.\+8.

On node version 0.\+6, it will remove all listeners on the input stream\textquotesingle{}s {\ttfamily data} and {\ttfamily keypress} events, because the readline module did not fully clean up after itself in that version of node, and did not make it possible to clean up after it in a way that has no potential for side effects.

Additionally, some of the readline options (like {\ttfamily terminal}) will not function in versions of node before 0.\+8, because they were not implemented in the builtin readline module.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4142}{}\doxysection{\texorpdfstring{CONTRIBUTING}{CONTRIBUTING}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__m4a8a320423d49ae259cbd52781d3011b_autotoc_md4142}
