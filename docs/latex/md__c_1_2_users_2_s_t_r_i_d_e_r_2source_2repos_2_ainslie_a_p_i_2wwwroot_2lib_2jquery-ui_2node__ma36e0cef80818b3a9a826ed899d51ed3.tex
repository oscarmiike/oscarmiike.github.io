\begin{quote}
Wrangle those bower dependencies and place each one where it\textquotesingle{}s supposed to be. \end{quote}

\begin{DoxyItemize}
\item Consistently positions your dependencies where you want them in your repository.
\item Conveniently facilitates \href{http://addyosmani.com/blog/checking-in-front-end-dependencies/}{\texttt{ tracking your dependencies}} without committing the entire Bower components folder.
\item Has the potential to reduce build times dramatically. For instance, if you were building a particular source folder for your webapp or website (compiling, concatenating, and minifying Java\+Script and CSS), you were forced to include the entire {\ttfamily bower\+\_\+components} directory in your build to make it work. This plugin clears the detritus.
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__ma36e0cef80818b3a9a826ed899d51ed3_autotoc_md8699}{}\doxysection{\texorpdfstring{Workflow}{Workflow}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__ma36e0cef80818b3a9a826ed899d51ed3_autotoc_md8699}
Whenever you add a new bower dependency, add which file should be copied and where to your Gruntfile {\ttfamily "{}bowercopy"{}} config. Then, run {\ttfamily grunt bowercopy}.

By default, bowercopy runs {\ttfamily bower install} for you (turn this off with the {\ttfamily runbower} option). Your bower directory is not removed so you can see which files you need from each component. It is suggested that you add the bower directory (usually `\textquotesingle{}bower\+\_\+components'{\ttfamily ) to your}.gitignore\`{}.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__ma36e0cef80818b3a9a826ed899d51ed3_autotoc_md8700}{}\doxysection{\texorpdfstring{Getting Started}{Getting Started}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__ma36e0cef80818b3a9a826ed899d51ed3_autotoc_md8700}
This plugin requires Grunt.

If you haven\textquotesingle{}t used \href{http://gruntjs.com/}{\texttt{ Grunt}} before, be sure to check out the \href{http://gruntjs.com/getting-started}{\texttt{ Getting Started}} guide, as it explains how to create a \href{http://gruntjs.com/sample-gruntfile}{\texttt{ Gruntfile}} as well as install and use Grunt plugins. Once you\textquotesingle{}re familiar with that process, you may install this plugin with this command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ grunt-\/bowercopy\ -\/-\/save-\/dev}

\end{DoxyCode}


Once the plugin has been installed, it may be enabled inside your Gruntfile with this line of Java\+Script\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{grunt.loadNpmTasks('grunt-\/bowercopy');}

\end{DoxyCode}
