 These are the built in themes that come with cardinal.

You can create more themes by copying and then editing the \href{https://github.com/thlorenz/cardinal/blob/master/themes/empty.js}{\texttt{ empty theme}} which is provided for that purpose.

The \href{https://github.com/thlorenz/cardinal/blob/master/themes/hide-semicolons.js}{\texttt{ hide semicolons theme}} has the added benefit of making all semicolons invisible. This makes code more readable at times.

Find out how to change the theme used by cardinal \href{https://github.com/thlorenz/cardinal\#theme}{\texttt{ here}}.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__mee03cb4017dd144a6251b91298533590_autotoc_md1888}{}\doxysection{\texorpdfstring{sharing themes}{sharing themes}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__mee03cb4017dd144a6251b91298533590_autotoc_md1888}
To add your theme to the cardinal built-\/in themes, follow the below steps\+:


\begin{DoxyEnumerate}
\item fork the cardinal repository
\item add your theme to the themes folder and commit your changes
\item create a pull request
\end{DoxyEnumerate}

If you believe that your theme is better than the current default theme, let me know by creating an issue.

This will allow others to cast their vote. If enough people agree, your theme will be promoted to be the default.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__mee03cb4017dd144a6251b91298533590_autotoc_md1889}{}\doxysubsection{\texorpdfstring{Contributed Themes}{Contributed Themes}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__mee03cb4017dd144a6251b91298533590_autotoc_md1889}
