\begin{DoxyVerb}uglifyjs [input files] [options]
\end{DoxyVerb}
 Uglify\+JS can take multiple input files. It\textquotesingle{}s recommended that you pass the input files first, then pass the options. Uglify\+JS will parse input files in sequence and apply any compression options. The files are parsed in the same global scope, that is, a reference from a file to some variable/function declared in another file will be matched properly.

If no input file is specified, Uglify\+JS will read from STDIN.

If you wish to pass your options before the input files, separate the two with a double dash to prevent input files being used as option arguments\+: \begin{DoxyVerb}uglifyjs --compress --mangle -- input.js
\end{DoxyVerb}

Otherwise the output goes to STDOUT.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__modules_2uglify-js_2_r_e_a_d_m_e_autotoc_md11897}{}\doxysubsection{\texorpdfstring{CLI source map options}{CLI source map options}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_ainslie_a_p_i_2wwwroot_2lib_2jquery-ui_2node__modules_2uglify-js_2_r_e_a_d_m_e_autotoc_md11897}
Uglify\+JS can generate a source map file, which is highly useful for debugging your compressed Java\+Script. To get a source map, pass {\ttfamily -\/-\/source-\/map -\/-\/output output.\+js} (source map will be written out to {\ttfamily output.\+js.\+map}).

Additional options\+:


\begin{DoxyItemize}
\item `-\/-\/source-\/map "{}filename='$<$\+NAME$>$\textquotesingle{}"{}{\ttfamily to specify the name of the source map. The value of }filename{\ttfamily is only used to set}file\`{} attribute (see \href{https://docs.google.com/document/d/1U1RGAehQwRypUTovF1KRlpiOFze0b-_2gc6fAH0KY0k}{\texttt{ the spec}}) in source map file.
\item `-\/-\/source-\/map "{}root='$<$\+URL$>$\textquotesingle{}"{}\`{} to pass the URL where the original files can be found.
\item {\ttfamily -\/-\/source-\/map "{}names=false"{}} to omit symbol names if you want to reduce size of the source map file.
\item `-\/-\/source-\/map "{}url='$<$\+URL$>$\textquotesingle{}"{}{\ttfamily to specify the URL where the source map can be found. Otherwise Uglify\+JS assumes HTTP}X-\/\+Source\+Map{\ttfamily is being used and will omit the }//\# source\+Mapping\+URL=\`{} directive.
\end{DoxyItemize}

For example\+: \begin{DoxyVerb}uglifyjs js/file1.js js/file2.js \
         -o foo.min.js -c -m \
         --source-map "root='http://foo.com/src',url='foo.min.js.map'"

The code generator tries to output shortest code possible by default. In case you want beautified output, pass {\ttfamily -\/-\/beautify} ({\ttfamily -\/b}). Optionally you can pass additional arguments that control the code output\+:


\begin{DoxyItemize}
\item {\ttfamily annotations} (default\+: {\ttfamily false}) — pass {\ttfamily true} to retain comment annotations {\ttfamily /\texorpdfstring{$\ast$}{*}@\+\_\+\+\_\+\+PURE\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/} or {\ttfamily /\texorpdfstring{$\ast$}{*}\#\+\_\+\+\_\+\+PURE\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/}, otherwise they will be discarded even if {\ttfamily comments} is set.

\item {\ttfamily beautify} (default\+: {\ttfamily true}) — whether to actually beautify the output. To allow for better optimizations, the compiler makes various assumptions\+:


\begin{DoxyItemize}
\item The code does not rely on preserving its runtime performance characteristics. Typically uglified code will run faster due to less instructions and easier inlining, but may be slower on rare occasions for a specific platform, e.\+g. see \`{}reduce\+\_\+funcs\`{}.
\item {\ttfamily .to\+String()} and {\ttfamily .value\+Of()} don\textquotesingle{}t have side effects, and for built-\/in objects they have not been overridden.
\item {\ttfamily undefined}, {\ttfamily NaN} and {\ttfamily Infinity} have not been externally redefined.
\item {\ttfamily arguments.\+callee}, {\ttfamily arguments.\+caller} and {\ttfamily Function.\+prototype.\+caller} are not used.
\item The code doesn\textquotesingle{}t expect the contents of {\ttfamily Function.\+prototype.\+to\+String()} or {\ttfamily Error.\+prototype.\+stack} to be anything in particular.
\item Getting and setting properties on a plain object does not cause other side effects (using {\ttfamily .watch()} or {\ttfamily Proxy}).
\item Object properties can be added, removed and modified (not prevented with {\ttfamily Object.\+define\+Property()}, {\ttfamily Object.\+define\+Properties()}, {\ttfamily Object.\+freeze()}, {\ttfamily Object.\+prevent\+Extensions()} or {\ttfamily Object.\+seal()}).
\item If array destructuring is present, index-\/like properties in {\ttfamily Array.\+prototype} have not been overridden\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Object.prototype[0]\ =\ 42;}
\DoxyCodeLine{var\ [\ a\ ]\ =\ [];}
\DoxyCodeLine{var\ \{\ 0:\ b\ \}\ =\ \{\};}
\DoxyCodeLine{//\ 42\ undefined}
\DoxyCodeLine{console.log([][0],\ a);}
\DoxyCodeLine{//\ 42\ 42}
\DoxyCodeLine{console.log(\{\}[0],\ b);}

\end{DoxyCode}

\item Earlier versions of Java\+Script will throw {\ttfamily Syntax\+Error} with the following\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{(\{}
\DoxyCodeLine{\ \ \ \ p:\ 42,}
\DoxyCodeLine{\ \ \ \ get\ p()\ \{\},}
\DoxyCodeLine{\});}
\DoxyCodeLine{//\ SyntaxError:\ Object\ literal\ may\ not\ have\ data\ and\ accessor\ property\ with}
\DoxyCodeLine{//\ \ \ \ \ \ \ \ \ \ \ \ \ \ the\ same\ name}

\end{DoxyCode}
