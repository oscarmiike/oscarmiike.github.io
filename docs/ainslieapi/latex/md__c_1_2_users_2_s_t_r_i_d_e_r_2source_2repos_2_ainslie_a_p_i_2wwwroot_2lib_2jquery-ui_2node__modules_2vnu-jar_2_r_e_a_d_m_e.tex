The Nu Html Checker (v.\+Nu) helps you \href{https://validator.w3.org/nu/about.html\#why-validate}{\texttt{ catch unintended mistakes in your HTML, CSS, and SVG}}. It enables you to \href{https://validator.github.io/validator/\#usage}{\texttt{ batch-\/check documents from the command line}} and from other scripts/apps, and to \href{https://validator.github.io/validator/\#standalone}{\texttt{ deploy your own instance of the checker as a service}} (like \href{https://validator.w3.org/nu/}{\texttt{ validator.\+w3.\+org/nu}}). Its \href{https://github.com/validator/validator}{\texttt{ source code is available}}, as are \href{https://validator.github.io/validator/\#build-instructions}{\texttt{ instructions on how to build, test, and run the code}}.

{\ttfamily vnu.\+jar} is a packaged version of the Nu Html Checker for batch-\/checking documents from the command line and from other scripts/apps.

Run the checker with one of the following invocations\+:

• {\ttfamily vnu-\/runtime-\/image/bin/vnu OPTIONS FILES} (Linux or mac\+OS)

• {\ttfamily vnu-\/runtime-\/image\textbackslash{}bin\textbackslash{}vnu.\+bat OPTIONS FILES} (Windows)

• {\ttfamily java -\/jar \texorpdfstring{$\sim$}{\string~}/vnu.jar OPTIONS FILES} (any system with Java8+ installed)

…where \+\_\+{\ttfamily FILES}\+\_\+ are the documents to check, and \+\_\+{\ttfamily OPTIONS}\+\_\+ are zero or more of the following options\+: \begin{DoxyVerb}--errors-only --Werror --exit-zero-always --stdout --asciiquotes
--user-agent USER_AGENT --no-langdetect --no-stream --filterfile FILENAME
--filterpattern PATTERN --css --skip-non-css --also-check-css --svg
--skip-non-svg --also-check-svg --html --skip-non-html
--format gnu|xml|json|text --help --verbose --version
\end{DoxyVerb}
 The \href{https://validator.github.io/validator/\#options}{\texttt{ Options}} section below provides details on each option, and the rest of this section provides some specific examples.

{\bfseries{Note\+:}} Throughout these examples, replace {\ttfamily \texorpdfstring{$\sim$}{\string~}/vnu.jar} with the actual path to that jar file on your system, and replace {\ttfamily vnu-\/runtime-\/image/bin/vnu} and {\ttfamily vnu-\/runtime-\/image\textbackslash{}bin\textbackslash{}vnu.\+bat} with the actual path to the {\ttfamily vnu} or {\ttfamily vnu.\+bat} program on your system — or if you add the {\ttfamily vnu-\/runtime-\/image/bin} or {\ttfamily vnu-\/runtime-\/image\textbackslash{}bin} directory your system {\ttfamily PATH} environment variable, you can invoke the checker with just {\ttfamily vnu}.

To check one or more documents from the command line\+: \begin{DoxyVerb}  vnu-runtime-image/bin/vnu      FILE.html FILE2.html FILE3.html...

  vnu-runtime-image\bin\vnu.bat  FILE.html FILE2.html FILE3.html...

  java -jar ~/vnu.jar            FILE.html FILE2.html FILE3.html...
\end{DoxyVerb}
 {\bfseries{Note\+:}} If you get a {\ttfamily Stack\+Overflow\+Error} error when invoking the checker, try adjusting the thread stack size by providing the {\ttfamily -\/Xss} option to java\+: \begin{DoxyVerb}  java -Xss512k -jar ~/vnu.jar ...
