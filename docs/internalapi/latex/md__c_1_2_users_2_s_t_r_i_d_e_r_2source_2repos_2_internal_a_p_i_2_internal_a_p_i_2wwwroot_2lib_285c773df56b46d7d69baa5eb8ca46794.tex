Positive patterns (e.\+g. {\ttfamily foo} or {\ttfamily \texorpdfstring{$\ast$}{*}}) add to the results, while negative patterns (e.\+g. {\ttfamily !foo}) subtract from the results.

Therefore a lone negation (e.\+g. `\mbox{[}'!foo\textquotesingle{}\mbox{]}{\ttfamily ) will never match anything – use}\mbox{[}\textquotesingle{}\texorpdfstring{$\ast$}{*}\textquotesingle{}, \textquotesingle{}!foo\textquotesingle{}\mbox{]}\`{} instead.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_285c773df56b46d7d69baa5eb8ca46794_autotoc_md10386}{}\doxysection{\texorpdfstring{Globbing patterns}{Globbing patterns}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_285c773df56b46d7d69baa5eb8ca46794_autotoc_md10386}

\begin{DoxyItemize}
\item {\ttfamily \texorpdfstring{$\ast$}{*}} matches any number of characters, but not {\ttfamily /}
\item {\ttfamily ?} matches a single character, but not {\ttfamily /}
\item {\ttfamily \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}} matches any number of characters, including {\ttfamily /}, as long as it\textquotesingle{}s the only thing in a path part
\item {\ttfamily \{\}} allows for a comma-\/separated list of "{}or"{} expressions
\item {\ttfamily !} at the beginning of a pattern will negate the match
