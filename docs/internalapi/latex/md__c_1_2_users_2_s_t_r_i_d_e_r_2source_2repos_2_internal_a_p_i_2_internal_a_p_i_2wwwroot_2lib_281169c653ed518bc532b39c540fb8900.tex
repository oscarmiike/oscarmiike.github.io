\begin{DoxyCode}{0}

{\bfseries{Note\+:}} The job of the {\ttfamily grunt} command is to load and run the version of Grunt you have installed locally to your project, irrespective of its version. Starting with Grunt v0.\+4, you should never install Grunt itself globally. For more information about why, \href{http://nodejs.org/en/blog/npm/npm-1-0-global-vs-local-installation}{\texttt{ please read this}}.

If you prefer the idiomatic Node.\+js method to get started with a project ({\ttfamily npm install \&\& npm test}) then install grunt-\/cli locally with {\ttfamily npm install grunt-\/cli -\/-\/save-\/dev}. Then add a script to your {\ttfamily package.\+json} to run the associated grunt command\+: {\ttfamily "{}scripts"{}\+: \{ "{}test"{}\+: "{}grunt test"{} \}}. Now {\ttfamily npm test} will use the locally installed {\ttfamily ./node\+\_\+modules/.bin/grunt} executable to run your Grunt commands.
