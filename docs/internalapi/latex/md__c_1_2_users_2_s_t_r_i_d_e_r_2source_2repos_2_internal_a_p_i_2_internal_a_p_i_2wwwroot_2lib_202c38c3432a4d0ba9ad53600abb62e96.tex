\begin{quote}
Terminal string styling done right \end{quote}
\href{https://travis-ci.org/chalk/chalk}{\texttt{ }} \href{https://coveralls.io/r/chalk/chalk?branch=master}{\texttt{ }} \href{https://www.youtube.com/watch?v=9auOCbH5Ns4}{\texttt{ }}

\href{https://github.com/Marak/colors.js}{\texttt{ colors.\+js}} used to be the most popular string styling module, but it has serious deficiencies like extending {\ttfamily String.\+prototype} which causes all kinds of \href{https://github.com/yeoman/yo/issues/68}{\texttt{ problems}}. Although there are other ones, they either do too much or not enough.

{\bfseries{Chalk is a clean and focused alternative.}}

Chalk comes with an easy to use composable API where you just chain and nest the styles you want.


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ chalk\ =\ require('chalk');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ style\ a\ string}
\DoxyCodeLine{chalk.blue('Hello\ world!');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ combine\ styled\ and\ normal\ strings}
\DoxyCodeLine{chalk.blue('Hello')\ +\ 'World'\ +\ chalk.red('!');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ compose\ multiple\ styles\ using\ the\ chainable\ API}
\DoxyCodeLine{chalk.blue.bgRed.bold('Hello\ world!');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ pass\ in\ multiple\ arguments}
\DoxyCodeLine{chalk.blue('Hello',\ 'World!',\ 'Foo',\ 'bar',\ 'biz',\ 'baz');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ nest\ styles}
\DoxyCodeLine{chalk.red('Hello',\ chalk.underline.bgBlue('world')\ +\ '!');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ nest\ styles\ of\ the\ same\ type\ even\ (color,\ underline,\ background)}
\DoxyCodeLine{chalk.green(}
\DoxyCodeLine{\ \ \ \ 'I\ am\ a\ green\ line\ '\ +}
\DoxyCodeLine{\ \ \ \ chalk.blue.underline.bold('with\ a\ blue\ substring')\ +}
\DoxyCodeLine{\ \ \ \ '\ that\ becomes\ green\ again!'}
\DoxyCodeLine{);}

\end{DoxyCode}


Easily define your own themes.


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ chalk\ =\ require('chalk');}
\DoxyCodeLine{var\ error\ =\ chalk.bold.red;}
\DoxyCodeLine{console.log(error('Error!'));}

\end{DoxyCode}

\DoxyCodeLine{var\ name\ =\ 'Sindre';}
\DoxyCodeLine{console.log(chalk.green('Hello\ \%s'),\ name);}
\DoxyCodeLine{//=>\ Hello\ Sindre}

\end{DoxyCode}

Example\+: `chalk.\+red.\+bold.\+underline(\textquotesingle{}Hello', \textquotesingle{}world\textquotesingle{});\`{}

Chain styles and call the last one as a method with a string argument. Order doesn\textquotesingle{}t matter, and later styles take precedent in case of a conflict. This simply means that {\ttfamily Chalk.\+red.\+yellow.\+green} is equivalent to {\ttfamily Chalk.\+green}.

Multiple arguments will be separated by space.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_202c38c3432a4d0ba9ad53600abb62e96_autotoc_md5489}{}\doxysubsubsection{\texorpdfstring{chalk.\+enabled}{chalk.\+enabled}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_202c38c3432a4d0ba9ad53600abb62e96_autotoc_md5489}
Color support is automatically detected, but you can override it by setting the {\ttfamily enabled} property. You should however only do this in your own code as it applies globally to all chalk consumers.

If you need to change this in a reusable module create a new instance\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ ctx\ =\ new\ chalk.constructor(\{enabled:\ false\});}

\end{DoxyCode}

Detect whether the terminal \href{https://github.com/chalk/supports-color}{\texttt{ supports color}}. Used internally and handled for you, but exposed for convenience.

Can be overridden by the user with the flags {\ttfamily -\/-\/color} and {\ttfamily -\/-\/no-\/color}. For situations where using {\ttfamily -\/-\/color} is not possible, add an environment variable {\ttfamily FORCE\+\_\+\+COLOR} with any value to force color. Trumps {\ttfamily -\/-\/no-\/color}.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_202c38c3432a4d0ba9ad53600abb62e96_autotoc_md5491}{}\doxysubsubsection{\texorpdfstring{chalk.\+styles}{chalk.\+styles}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_202c38c3432a4d0ba9ad53600abb62e96_autotoc_md5491}
Exposes the styles as \href{https://github.com/chalk/ansi-styles}{\texttt{ ANSI escape codes}}.

Generally not useful, but you might need just the {\ttfamily .open} or {\ttfamily .close} escape code if you\textquotesingle{}re mixing externally styled strings with your own.

Chalk does not support anything other than the base eight colors, which guarantees it will work on all terminals and systems. Some terminals, specifically {\ttfamily xterm} compliant ones, will support the full range of 8-\/bit colors. For this the lower level \href{https://github.com/jbnicolai/ansi-256-colors}{\texttt{ ansi-\/256-\/colors}} package can be used.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_202c38c3432a4d0ba9ad53600abb62e96_autotoc_md5499}{}\doxysubsection{\texorpdfstring{Windows}{Windows}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_202c38c3432a4d0ba9ad53600abb62e96_autotoc_md5499}
