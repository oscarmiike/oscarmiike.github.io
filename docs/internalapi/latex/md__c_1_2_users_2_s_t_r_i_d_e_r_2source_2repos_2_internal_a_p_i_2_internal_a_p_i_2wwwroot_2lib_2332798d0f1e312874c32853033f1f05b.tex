{\bfseries{clone}} masters cloning simple objects (even with custom prototype), arrays, Date objects, and Reg\+Exp objects. Everything is cloned recursively, so that you can clone dates in arrays in objects, for example.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_2332798d0f1e312874c32853033f1f05b_autotoc_md5505}{}\doxysection{\texorpdfstring{API}{API}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_2332798d0f1e312874c32853033f1f05b_autotoc_md5505}
{\ttfamily clone(val, circular, depth)}


\begin{DoxyItemize}
\item {\ttfamily val} -- the value that you want to clone, any type allowed
\item {\ttfamily circular} -- boolean

Call {\ttfamily clone} with {\ttfamily circular} set to {\ttfamily false} if you are certain that {\ttfamily obj} contains no circular references. This will give better performance if needed. There is no error if {\ttfamily undefined} or {\ttfamily null} is passed as {\ttfamily obj}.
\item {\ttfamily depth} -- depth to which the object is to be cloned (optional, defaults to infinity)
\item {\ttfamily prototype} -- sets the prototype to be used when cloning an object. (optional, defaults to parent prototype).
\item {\ttfamily include\+Non\+Enumerable} -- set to {\ttfamily true} if the non-\/enumerable properties should be cloned as well. Non-\/enumerable properties on the prototype chain will be ignored. (optional, defaults to {\ttfamily false})
\end{DoxyItemize}

{\ttfamily clone.\+clone\+Prototype(obj)}


\begin{DoxyItemize}
\item {\ttfamily obj} -- the object that you want to clone
\end{DoxyItemize}
