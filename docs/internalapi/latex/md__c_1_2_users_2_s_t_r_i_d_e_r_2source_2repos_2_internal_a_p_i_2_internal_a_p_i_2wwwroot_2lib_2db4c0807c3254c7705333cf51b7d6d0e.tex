Install this grunt plugin next to your project\textquotesingle{}s \href{https://gruntjs.com/getting-started}{\texttt{ Gruntfile.\+js}} with\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ grunt-\/html\ -\/-\/save-\/dev}

\end{DoxyCode}


Then add this line to your project\textquotesingle{}s {\ttfamily Gruntfile.\+js}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{grunt.loadNpmTasks('grunt-\/html');}

\end{DoxyCode}


Then specify what files to validate in your config\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{grunt.initConfig(\{}
\DoxyCodeLine{\ \ htmllint:\ \{}
\DoxyCodeLine{\ \ \ \ all:\ ['demos/**/*.html',\ 'tests/**/*.html']}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{\});}

\end{DoxyCode}


For fast validation, keep that in a single group, as the validator initialization takes a few seconds.

When combined with a watching task (such as \href{https://github.com/gruntjs/grunt-contrib-watch}{\texttt{ grunt-\/contrib-\/watch}}), even faster validation can be achieved by starting the validator in client mode and connecting to an already-\/running instance of the validator in server mode. This removes the time required by repeated initializations. See the {\ttfamily server} option below.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_2db4c0807c3254c7705333cf51b7d6d0e_autotoc_md9085}{}\doxysection{\texorpdfstring{Options}{Options}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_2db4c0807c3254c7705333cf51b7d6d0e_autotoc_md9085}

When {\ttfamily server} is set to a falsy value, the validator is invoked using {\ttfamily java -\/jar}, which can be considered normal operation.

Set {\ttfamily server} to an object to start the validator in client mode and connect to an already-\/running instance of the validator in server mode. To start the validator in server mode, use {\ttfamily java -\/cp "{}path/to/vnu.\+jar"{} nu.\+validator.\+servlet.\+Main \texorpdfstring{$<$}{<}port\texorpdfstring{$>$}{>}}.
