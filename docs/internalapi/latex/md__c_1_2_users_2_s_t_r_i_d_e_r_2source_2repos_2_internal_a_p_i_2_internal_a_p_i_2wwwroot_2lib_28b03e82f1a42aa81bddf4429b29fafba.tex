 A backwards-\/compatible fs.\+realpath for Node v6 and above

In Node v6, the Java\+Script implementation of fs.\+realpath was replaced with a faster (but less resilient) native implementation. That raises new and platform-\/specific errors and cannot handle long or excessively symlink-\/looping paths.

This module handles those cases by detecting the new errors and falling back to the Java\+Script implementation. On versions of Node prior to v6, it has no effect.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_28b03e82f1a42aa81bddf4429b29fafba_autotoc_md2371}{}\doxysection{\texorpdfstring{USAGE}{USAGE}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_28b03e82f1a42aa81bddf4429b29fafba_autotoc_md2371}
