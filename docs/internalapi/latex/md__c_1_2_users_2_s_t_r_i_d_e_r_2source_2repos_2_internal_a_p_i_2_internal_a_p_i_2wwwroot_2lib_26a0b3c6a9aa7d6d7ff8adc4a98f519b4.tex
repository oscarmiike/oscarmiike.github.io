 \begin{quote}
Resolve the path of a package regardless of it having an entry point \end{quote}
Some packages like CLI tools and grunt tasks don\textquotesingle{}t have a entry point, like {\ttfamily "{}main"{}\+: "{}foo.\+js"{}} in package.\+json, resulting in them not being resolvable by {\ttfamily require.\+resolve()}. Unlike {\ttfamily require.\+resolve()}, this module also resolves packages without an entry point, returns {\ttfamily undefined} instead of throwing when the module can\textquotesingle{}t be found, and resolves from {\ttfamily process.\+cwd()} instead {\ttfamily \+\_\+\+\_\+dirname} by default.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_26a0b3c6a9aa7d6d7ff8adc4a98f519b4_autotoc_md11219}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_26a0b3c6a9aa7d6d7ff8adc4a98f519b4_autotoc_md11219}
