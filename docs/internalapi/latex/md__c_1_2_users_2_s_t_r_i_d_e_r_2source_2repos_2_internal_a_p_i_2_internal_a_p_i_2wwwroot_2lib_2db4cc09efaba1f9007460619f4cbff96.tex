 Implements a function similar to {\ttfamily performance.\+now} (based on {\ttfamily process.\+hrtime}).

Modern browsers have a {\ttfamily window.\+performance} object with -\/ among others -\/ a {\ttfamily now} method which gives time in miliseconds, but with sub-\/milisecond precision. This module offers the same function based on the Node.\+js native {\ttfamily process.\+hrtime} function.

According to the \href{http://www.w3.org/TR/hr-time/}{\texttt{ High Resolution Time specification}}, the number of miliseconds reported by {\ttfamily performance.\+now} should be relative to the value of {\ttfamily performance.\+timing.\+navigation\+Start}. For this module, it\textquotesingle{}s relative to when the time when this module got loaded. Right after requiring this module for the first time, the reported time is expected to have a near-\/zero value.

Using {\ttfamily process.\+hrtime} means that the reported time will be monotonically increasing, and not subject to clock-\/drift.\hypertarget{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_2db4cc09efaba1f9007460619f4cbff96_autotoc_md3974}{}\doxysection{\texorpdfstring{Example usage}{Example usage}}\label{md__c_1_2_users_2_s_t_r_i_d_e_r_2source_2repos_2_internal_a_p_i_2_internal_a_p_i_2wwwroot_2lib_2db4cc09efaba1f9007460619f4cbff96_autotoc_md3974}
