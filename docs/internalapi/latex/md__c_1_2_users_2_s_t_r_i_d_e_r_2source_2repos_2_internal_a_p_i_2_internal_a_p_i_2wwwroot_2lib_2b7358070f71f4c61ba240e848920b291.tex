{\itshape Run this task with the {\ttfamily grunt qunit} command.}

You have chosen to write your unit tests using \href{https://qunitjs.com/}{\texttt{ QUnit}}, you have written a html page which reports the summary and individual details of your unit tests, you are happy with this but realize you miss the ability to have your unit test suite run automatically each time you commit changes to your code.

This is where the {\ttfamily grunt-\/contrib-\/qunit} plugin comes in the play\+: {\ttfamily grunt-\/contrib-\/qunit} lets you run your tests in the invisible \mbox{[}Chrome\mbox{]}\mbox{[}\mbox{]} browser, thus converting your unit test suite into something you can run from a script, a script you can have automatically run on travis-\/ci (or the Continuous Integration service of your choice) which in turn can alert you of any rule-\/breaking commit to your code.

You can still monitor the status of your unit tests suite visiting your html test page in your browser, but with {\ttfamily grunt-\/contrib-\/qunit} you can also run the same suite from the command line interface.

This plugin defines one single task\+: {\ttfamily qunit}. Configure it in your {\ttfamily Gruntfile.\+js}, run it with the {\ttfamily grunt qunit} command.

Please read about specifying task targets, files and options in the grunt \href{https://gruntjs.com/configuring-tasks}{\texttt{ Configuring tasks}} guide.

When installed by npm, this plugin will automatically download and install a local \mbox{[}Chrome\mbox{]}\mbox{[}\mbox{]} binary within the {\ttfamily node\+\_\+modules} directory of the \href{https://pptr.dev/}{\texttt{ puppeteer}} library, which is used for launching a Chrome process. If your system already provides an installation of Chrome, you can configure this plugin to use the globally installed executable by specifying a custom {\ttfamily executable\+Path} in the puppeteer launch options. ~\newline

It\textquotesingle{}s important to note that grunt does not automatically start a {\ttfamily localhost} web server. That being said, the \href{https://github.com/gruntjs/grunt-contrib-connect}{\texttt{ grunt-\/contrib-\/connect plugin}} {\ttfamily connect} task can be run before the {\ttfamily qunit} task to serve files via a simple \href{http://www.senchalabs.org/connect/}{\texttt{ connect}} web server.

In the following example, if a web server isn\textquotesingle{}t running at {\ttfamily localhost\+:8000}, running {\ttfamily grunt qunit} with the following configuration will fail because the {\ttfamily qunit} task won\textquotesingle{}t be able to load the specified URLs. However, running {\ttfamily grunt connect qunit} will first start a static \href{http://www.senchalabs.org/connect/}{\texttt{ connect}} web server at {\ttfamily localhost\+:8000} with its base path set to the Gruntfile\textquotesingle{}s directory. Then, the {\ttfamily qunit} task will be run, requesting the specified URLs.

\href{http://api.qunitjs.com/category/callbacks/}{\texttt{ QUnit callback}} methods and arguments are also emitted through grunt\textquotesingle{}s event system so that you may build custom reporting tools. Please refer to to the QUnit documentation for more information.

The events, with arguments, are as follows\+:


\begin{DoxyItemize}
\item {\ttfamily qunit.\+begin}
\item {\ttfamily qunit.\+module\+Start} {\ttfamily (name)}
\item {\ttfamily qunit.\+test\+Start} {\ttfamily (name)}
\item {\ttfamily qunit.\+log} {\ttfamily (result, actual, expected, message, source)}
\item {\ttfamily qunit.\+test\+Done} {\ttfamily (name, failed, passed, total, duration)}
\item {\ttfamily qunit.\+module\+Done} {\ttfamily (name, failed, passed, total)}
\item {\ttfamily qunit.\+done} {\ttfamily (failed, passed, total, runtime)}
\end{DoxyItemize}

In addition to QUnit callback-\/named events, the following events are emitted by Grunt\+:


\begin{DoxyItemize}
\item {\ttfamily qunit.\+spawn} {\ttfamily (url)}\+: when \mbox{[}Chrome\mbox{]}\mbox{[}\mbox{]} is spawned for a test
\item {\ttfamily qunit.\+fail.\+load} {\ttfamily (url)}\+: when \mbox{[}Chrome\mbox{]}\mbox{[}\mbox{]} could not open the given url
\item {\ttfamily qunit.\+fail.\+timeout}\+: when a QUnit test times out, usually due to a missing {\ttfamily QUnit.\+start()} call
\item {\ttfamily qunit.\+error.\+on\+Error} {\ttfamily (message, stack\+Trace)}\+: when a Java\+Script execution error occurs
\end{DoxyItemize}
